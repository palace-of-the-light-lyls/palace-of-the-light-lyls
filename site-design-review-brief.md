# 光之殿堂網站設計評估 Brief

> 用途：提供給 Claude / GPT 評估目前網站的「網頁版」與「手機版」設計，請它們提出可執行的調整建議。  
> 專案性質：龍巖「光之殿堂」商品介紹 / 導覽網站，目前以靜態網站方式製作。  
> 評估重點：不是單純看美不美，而是要判斷資訊是否好理解、頁面節奏是否順、是否符合業務展示與消費者理解。

---

## 1. 網站目標

這個網站是「光之殿堂」的商品介紹與導覽輔助網站，主要用來協助業務或介紹者向使用者說明：

- 光之殿堂是什麼樣的紀念建築與自然場域
- 安藤忠雄的建築理念如何對應到光之殿堂
- 光、水、自然、幾何秩序等建築特色如何落在空間中
- 六大商品系列各自的設計概念、商品特色、團隊工藝、規格與樓層配置
- 使用者能透過圖像、配置圖與文字，理解每個商品系列的定位與差異

網站希望呈現的感受：

- 莊重、安定、溫柔
- 有建築感，但不能過度艱澀
- 有商品介紹功能，但不要像死板資料表
- 能服務業務展示，也能讓一般消費者看懂
- 視覺上要有高級感、留白與節奏，但不能讓資訊變得難找

---

## 2. 目標受眾

主要受眾可分為三類：

1. 業務人員  
   用網站輔助介紹商品、空間特色、設計故事與配置關係。

2. 授權經銷商 / 內部介紹者  
   需要快速理解各商品系列差異，並能轉述給客戶。

3. 一般消費者 / 家屬  
   需要在不熟悉建築語彙與商品規格的情況下，理解光之殿堂的價值與適合性。

因此評估時，請不要只用工程或設計師視角，也要用「行銷企劃」與「實際展示情境」來看。

## 3. 目前網站技術與形式

目前是靜態網站：

- `index.html`
- `styles.css`
- `script.js`
- 圖片與影片素材放在 `assets/`

目前互動功能：

- 頂部主選單切換頁面

- URL hash 可切換頁面與商品，例如 `#series-water`
- 樓層與平面配置圖可點擊放大
- 部分頁面有桌機文案與手機文案分開處理：`.desktop-copy` / `.mobile-copy`

## 10. 樓層與平面配置目前設計

目前每個商品的「樓層與平面配置」大致包含：

- 標題：樓層與平面配置
- 小標籤：一樓 / 二樓
- 一段簡短說明
- 商品區位圖
- 空間區位圖
- 容量 / 區域數據
- 圖面備註：此為規劃階段圖面，最終尺寸圖說請以核定建照圖為準

目前配置圖邏輯：

- 大多數商品位於一樓
- 大觀位於二樓
- 商品區位圖通常共用同一張一樓配置圖，只是黃色標示區域不同
- 空間區位圖依商品不同而不同

目前已知設計要求：

- 配置圖不能只是丟圖，要讓使用者知道「這張圖想表達什麼」
- 說明文字要站在網站行銷企劃角度，不要像教科書或死板資訊表
- 商品規格、數量、樓層配置等資訊都需要有存在意義
- 不能只是堆資料，應該幫助使用者理解商品位置、空間感與價值

希望評估：

- 配置圖區塊目前是否好理解？
- 「商品區位圖」與「空間區位圖」的差異是否清楚？
- 是否需要新增更行銷式的圖說？
- 樓層標籤「一樓 / 二樓」是否夠明顯？
- 容量數據是否應該保留在配置區，還是只留在容量配置表？

- 可以更有質感
- 可以增加互動
- 可以優化手機版
- 可以讓文字更精簡

請改成具體建議，例如：

- 哪一頁哪一段需要縮短
- 哪個區塊應該上移或下移
- 哪張圖與哪段文字不對應
- 哪個手機版區塊應該改成折疊、輪播、單欄或分段
- 哪些資料表應該保留，哪些應該改成摘要卡片

## 16. 補充限制

目前不希望大改網站架構或換框架。建議請以低維護、可交接、可逐步修改為主。

優先順序應該是：

1. 使用者看得懂
2. 業務好展示
3. 商品差異清楚
4. 視覺質感一致
5. 技術維護簡單

請以「網站行銷企劃 + UX 審稿」的角度評估，而不是只從工程或美術角度評估。
